# MyGrep - Распределённая CLI-утилита grep

Данный проект реализует распределённую версию классической утилиты `grep`. Координатор разбивает входной файл на части (чанки) и распределяет их между воркерами. Результаты собираются и выводятся после обработки всех чанков при условии соблюдения кворума.

## Особенности
- Распределённая обработка данных.
- Поддержка кворума (вывод результатов только если N/2+1 серверов ответили).
- Передача флагов поиска (`-i`, `-v`, `-B`, `-A`, `-C`, `-F`) от координатора к воркерам.
- Параллелизм через горутины и каналы.

## Быстрый старт

### 1. Сборка
```bash
go build -o coordinator.exe coordinator.go
go build -o worker.exe worker.go
```

### 2. Запуск воркеров
Запустите несколько экземпляров воркеров на разных портах:
```powershell
Start-Process .\worker.exe -ArgumentList "-port :6001 -callback http://localhost:6000"
Start-Process .\worker.exe -ArgumentList "-port :6002 -callback http://localhost:6000"
```

### 3. Запуск координатора
Координатор распределит задачи по указанным серверам:
```bash
.\coordinator.exe --pattern "ERROR" --servers "http://localhost:6001,http://localhost:6002" --file testdata.txt --quorum 2 --i
```

## Сравнительные тесты
Для автоматического сравнения вывода `mygrep` с эталонным `grep` предусмотрены скрипты для разных сред.

### PowerShell (Windows)
```powershell
.\compare_test.ps1
```

### Bash (Linux/macOS)
```bash
chmod +x compare_test.sh
./compare_test.sh
```

Результаты тестирования показывают 100% соответствие вывода `mygrep` и стандартного `grep` для базовых операций, поиска без учета регистра, инвертированного поиска и поиска с контекстом.
